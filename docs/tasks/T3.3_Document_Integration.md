## Task ID
T3.3

## Layer

`feature/T3.3-document-integration-l1`

## Goal

Confirm that the **Document API (T3.1a–d)** and **Document UI (T3.2a–b)** work together end-to-end, and update all
relevant documentation to reflect the Document slice as a first-class, integrated feature.

## Prerequisites

- Backend:
  - T3.1a–d complete and merged (Document API L0–L3).
- Frontend:
  - T3.2a–b complete and merged (Document UI L0–L1).
- All tests/builds are green on main.

## Requirements

As DocsAgent, you MUST NOT modify backend/ or frontend/ code. You operate only in `docs/`.

### 1. End-to-End Verification Checklist (DEV_ONBOARDING.md)

Add a “Document end-to-end verification checklist”, similar to Claims, including:

- Ensure Neo4j dev/test is running and configured via `backend/.env`.
- Start backend:

  ```bash
  cargo run --manifest-path backend/Cargo.toml
  ```

- Start frontend:

  ```bash
  cd frontend
  npm run dev
  ```

- Visit `/documents` in the browser and verify:
  - Loading → success states.
  - Empty → populated list (after inserting test data).
  - Error behavior when backend is down.

- Optional:
  - `cargo test --tests --manifest-path backend/Cargo.toml`
  - `npm run test` and `npm run build`
