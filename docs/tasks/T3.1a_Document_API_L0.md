# T3.1a — Document API L0 (Skeleton + Stubs)

## Task ID
T3.1a

## Layer
L0 (skeleton, compile-only, no real DB)

## Persona
BackendAgent

## Branch
`feature/T3.1a-document-api-l0`

## Goal

Introduce a **minimal Document API skeleton** and DTOs so that the backend compiles and exposes a stubbed
`GET /documents` endpoint, without any Neo4j logic yet. This mirrors T2.1a for Claims.

## Requirements

### 1. DTOs and Models

- Add a `DocumentDto` in `backend/src/dto/document.rs` (or in `dto/mod.rs` if you keep DTOs flat) with fields:

  ```rust
  pub struct DocumentDto {
      pub id: String,
      pub title: String,
      pub doc_type: String,       // e.g. "pdf", "motion", "ruling", "evidence", "filing"
      pub created_at: Option<String>, // ISO-8601 string or None
  }
  ```

- Add a `Document` model struct in `backend/src/models/document.rs`. This can be minimal at L0 and expanded later.

- Export the DTO and model appropriately in `backend/src/dto/mod.rs` and `backend/src/models/mod.rs` if needed.

### 2. API Module

- Create a new module `backend/src/api/documents.rs` that defines a stub handler:

  ```rust
  pub async fn list_documents() -> impl IntoResponse { ... }
  ```

- The handler should return a stubbed list of `DocumentDto` values (hard-coded in memory).

- No Neo4j calls at this layer.

### 3. Router Wiring

- Update `backend/src/api/mod.rs` to:

  - Declare the `documents` module.
  - Register a route `GET /documents` pointing to `documents::list_documents`.

### 4. Compile Cleanly

- `cargo check --manifest-path backend/Cargo.toml` must succeed.
- No unused imports, dead code warnings, or missing module references related to the new Document code.

## Out of Scope

- No Neo4j logic.
- No tests are strictly required at L0.
- No frontend work.
- No changes to existing Claims API.

## Acceptance Criteria

- Backend compiles cleanly.
- `GET /documents` exists at the router level and returns stubbed JSON that matches `DocumentDto`.
- No runtime panics from the new code paths.

## Tests

- Optional for this task (L0). You may add a very small unit test for the handler if desired, but it is not required.

## Notes

- This is the Document equivalent of T2.1a for Claims.
- Later tasks (T3.1b–d) will add real Neo4j logic, validation, and analysis.
