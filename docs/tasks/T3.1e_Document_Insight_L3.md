# T3.1e — Document Insight L3 (GET /documents/recent)
Layer: L3  
Persona: BackendAgent  
Branch: feature/T3.1e-document-insight-l3  

## 1. Purpose

Introduce the first Document insight endpoint for the MVP.
This endpoint exposes recent document activity so the UI can highlight
“What was added to the case most recently?”

## 2. Requirements

### 2.1 New Endpoint
Implement:
GET /documents/recent

### 2.2 Behavior
- Return the N most recently ingested documents, sorted by `ingested_at DESC`
- Default `N = 10`
- Optional: GET /documents/recent?limit=20
- Only include documents where `ingested_at` is NOT NULL
- Response: JSON array of DocumentDto

### 2.3 Data Model Alignment
Document fields from DATA_MODEL.md:
- id, title, type, file_path
- created_at
- ingested_at (optional, datetime)

No new fields may be added.

## 3. Implementation Requirements

### 3.1 Repository Method
Add to backend/src/repositories/document_repository.rs:

pub async fn list_recent_documents(&self, limit: i64) -> Result<Vec<Document>, RepoError>;

Cypher:

MATCH (d:Document)
WHERE d.ingested_at IS NOT NULL
RETURN d
ORDER BY d.ingested_at DESC
LIMIT $limit

### 3.2 API Handler
Add in backend/src/api/documents.rs:

- Handler: list_recent_documents
- Parse limit query param (default 10)
- Call repository
- Map domain → DTO
- Return 200 OK

### 3.3 Router
In backend/src/api/mod.rs:

.route("/documents/recent", get(documents::list_recent_documents))

### 3.4 Tests (Required for L3)
Create backend/tests/test_documents_recent.rs:

- Insert test documents (some with ingested_at, some without)
- Call GET /documents/recent
- Assert:
  - Sorted by ingested_at DESC
  - Limit honored
  - Only docs with ingested_at returned
- Use markers: source="test", test_run_id

## 4. Acceptance Criteria
